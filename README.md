# SGHG-SE

## Overview

```SGHG-SE``` is a model that integrates Dual-view representation learning and the Squeeze-and-Excitation (SE) mechanism for predicting missing reactions in genome-scale metabolic models (GEMs). For draft GEMs generated by automated reconstruction pipelines, SGHG-SE can be used to further conduct gap-fillings by predicting confidence scores of candidate metabolic reactions. 

SGHG-SE consists of three modules: the feature extraction module, the feature enhancement module, and the output module. The feature extraction module adopts a dual-view representation learning architecture that combines a sparse graph convolutional neural network and a hypergraph network to extract the topological structural feature representations of metabolites. The feature enhancement module integrates the SE mechanism to distinguish specific substrates from common cofactors among metabolites, and improves the prediction performance by enhancing or suppressing their feature densities, respectively. The output module aggregates metabolite features into reaction features and outputs the prediction scores of candidate reactions via a multi-layer perceptron (MLP).

## Description
There are totally five folders, which are briefly described as below.   
- The folder ```Data``` contains the datasets for training and testing SGHG-SE, and contains the summarized experimental results that are also available in the supplementary materials.
- The folder ```SGHG-SE_prediction``` contains the codes and corresponding files for the evaluation of reaction prediction performances. 
- The folder ```SGHG-SE_recovery``` contains the codes and corresponding files for the evaluation of reaction recovery.
- The folder ```SGHG-SE_ablation``` contains the codes and corresponding files for the ablation study.
- The folder ```SGHG-SE_gapfilling``` contains the codes and corresponding files for gap-fillings and metabolic phenotype predictions.

## Usage

### Datasets 

- The folder ```Data/Dataset/BiGG dataset``` contains 108 BiGG GEMs, which are also available in other sources http://bigg.ucsd.edu/models.
- The folder ```Data/Dataset/Taxonomy dataset``` contains 24 draft GEMs, where are generated by CarveMe from the 24 bacterial genomes https://www.ncbi.nlm.nih.gov/datasets/genome.
- The folder ```Data/Dataset/Reaction pool``` contains the universal reaction pool, which is also available in other sources https://github.com/canc1993/cheshire-gapfilling/tree/main/data/pools.

### Performance Evaluation of Reaction Prediction

- The codes are available in the folder ```SGHG-SE_prediction``` for evaluting reaction prediction performances of SGHG-SE.
- We can run the script ```SGHG-SE_prediction/main.py``` to conduct 5-fold cross-validations for evaluting reaction prediction performances of SGHG-SE on 108 GEMs. 
- The folder ```Data/Trained model/Trained_model (prediction)/iBWG_1329``` contains the trained HGLMA models for 5-fold cross-validations on iBWG_1329 (one of the 108 GEMs). 
- The evaluation results of prediction performances by SGHG-SE are summarized and available in ```Data/Performance Evaluation of Reaction Prediction```.

### Performance Evaluation of Reaction Recovery

- The codes are available in the folder ```SGHG-SE_recovery``` for evaluting reaction recovery performances of SGHG-SE.
- We can run the script ```SGHG-SE_recovery/main.py``` to evaluate reaction recovery performances of SGHG-SE on 108 GEMs. 
- The folder ```Data/Trained model/Trained_model (recovery)/iBWG_1329``` contains the trained SGHG-SE models for 5-fold cross-validations on iBWG_1329 (one of the 108 GEMs). 
- The evaluation results of reaction recovery performances by SGHG-SE are summarized and available in ```Data/Performance Evaluation of Reaction Recovery```.



### Ablation Experiment

- The codes in the folder ```SGHG-SE_ablation``` correspond to three variants of HGLMA for the ablation study. 
- We can run the script ```SGHG-SE_ablation/SGHG-SE/main.py``` to conduct 5-fold cross-validations  of the variant model on 108 GEMs.
- The results of the ablation study are summarized and available in  ```Data/Ablation Experiment```.
